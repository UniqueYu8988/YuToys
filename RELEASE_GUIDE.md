# 📦 YuToys V1.6.1 发布指南 (Perfect Release)

主人，这份指南将辅助您完成“小羽桌面助理”的完美发布，确保交付给其他使用者的是一个纯净、高效且充满仪式感的版本。

## 1. 核心构建命令

在项目根目录打开终端，运行以下命令生成正式版 EXE：

```bash
# 执行打包
npm run build
```

打包完成后，您会在项目根目录看到一个 **`dist`** 或 **`release`** 文件夹。

## 2. 隐私数据清理 (非常重要喵！)

在准备发布包之前，请务必确认以下文件**没有**被包含在交付文件夹中：

- **UserData (localStorage)**：
  - 小羽的数据存储在 Windows 的 AppData 目录下（通常是 `%APPDATA%/YuToys`）。
  - **放心喵**：如果您只是打包 EXE 发给别人，别人的电脑启动时会自动创建一个空的本地存储，**不会**带走您的任务列表和运势记录。
- **.env 文件**：如果您在开发中使用了敏感的 API Key，请确保构建出的包中不含有 `.env` 系列文件。
- **源码文件**：发布时仅需交付打包后的安装程序或绿色版 EXE 文件夹，无需包含 `src/` 或 `node_modules/`。

## 3. GitHub Release 交付建议

在 GitHub 仓库页面点击 **“Create a new release”**，建议按如下规格填写：

- **Tag version**: `v1.6.1`
- **Release title**: `小羽桌面助理 V1.6.1 - 完美仪式感版`
- **Description**: 直接粘贴我们新写的 [README.md](./README.md) 中的特色巡礼内容。
- **Assets**: 上传打包出的 `.exe` 或 `.zip` 压缩包。

## 4. 交付清单

一个完美的发布包应包含：

1.  **YuToys Setup 1.6.1.exe** (安装版) 或 **YuToys-win32-x64** (绿色免安装文件夹)
2.  一份精美的 **README.md** (引导使用者通过小羽开启新的一天)
